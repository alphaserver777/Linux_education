
# Практика по Linux

## Операции с файлами и каталогами

Темы:

* `cp` — копирование
* `mv` — перемещение и переименование
* `mkdir` — создание каталогов
* `rm` — удаление
* `ln` — ссылки

От простого к более сложному.

---

# Уровень 1 — Создание каталогов

## Задание 1

### Нужно:

1. Перейти в домашнюю директорию

2. Создать папку `practice`

3. Внутри создать:

* `docs`
* `images`
* `backup`

4. Проверить структуру

---

## Решение

```bash
cd ~

mkdir practice

mkdir practice/docs
mkdir practice/images
mkdir practice/backup

tree practice
```

Если `tree` нет:

```bash
ls -R practice
```

# Уровень 4 — Копирование каталогов

## Задание 4

### Нужно:

Скопировать папку `images` вместе со всем содержимым в `backup`

---

## Решение

```bash

```

### Объяснение:

* `-r` = recursive
Копирование папки со всем содержимым

# Объяснение простыми словами

## Symbolic link (soft link)

Это как ярлык в Windows.

Если удалить оригинал:

* ссылка сломается

---

## Hard link

Это ещё одно имя файла.

Если удалить оригинал:

* файл останется доступен через hard link

# Уровень 10 — DevOps практика

## Задание 10

Создать структуру:

```text
infra/
├── nginx/
│   └── nginx.conf
├── docker/
│   └── Dockerfile
├── kubernetes/
│   └── deployment.yaml
└── backup/
```

После:

1. Скопировать все конфиги в `backup`
2. Переименовать `deployment.yaml` в `k8s-deployment.yaml`
3. Удалить папку `docker`

---

## Решение

```bash
mkdir -p infra/{nginx,docker,kubernetes,backup}

touch infra/nginx/nginx.conf
touch infra/docker/Dockerfile
touch infra/kubernetes/deployment.yaml

cp infra/nginx/nginx.conf infra/backup/

cp infra/kubernetes/deployment.yaml infra/backup/

mv infra/kubernetes/deployment.yaml \
       infra/kubernetes/k8s-deployment.yaml

       rm -r infra/docker
       ```

# Уровень 11 — Мини-квест

## Ситуация

       Ты — junior DevOps.

       Нужно:

       1. Создать папку проекта
       2. Сделать backup конфигов
       3. Создать symbolic link на конфиг nginx
       4. Переместить backup в `/tmp`
       5. Удалить старые файлы

       ---

## Возможное решение

       ```bash
       mkdir -p project/nginx

       touch project/nginx/nginx.conf

       mkdir project/backup

       cp project/nginx/nginx.conf project/backup/

       ln -s project/nginx/nginx.conf project/nginx_link

       mv project/backup /tmp/

       rm project/nginx/nginx.conf
       ```

# Полезные команды

       | Команда | Что делает                  |
       | ------- | --------------------------- |
       | `cp`    | Копировать                  |
       | `cp -r` | Копировать папку            |
       | `mv`    | Переместить / переименовать |
       | `rm`    | Удалить файл                |
       | `rm -r` | Удалить папку               |
       | `mkdir` | Создать папку               |
       | `touch` | Создать файл                |
       | `ln`    | Hard link                   |
       | `ln -s` | Symbolic link               |

       ---

# Вопросы для проверки

## Что делает команда?

       ```bash
       mv file1 file2
       ```

       Ответ:
       Переименовывает файл `file1` в `file2`.

       ---

## Чем отличается:

       ```bash
       rm
       ```

       от

       ```bash
       rm -r
       ```

       Ответ:

       * `rm` удаляет файлы
       * `rm -r` удаляет папки и всё внутри

       ---

## Что опаснее?

       ```bash
       rm -rf /
       ```

       Ответ:
       Удаление всей системы Linux.
       Одна из самых опасных команд.
