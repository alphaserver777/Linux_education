# Теоретическая часть

Перед выполнением команды Bash сначала обрабатывает специальные конструкции.

Например:

```bash
echo *.txt
```

Сначала оболочка найдёт все файлы `.txt`, а затем передаст их команде `echo`.

---

# Основные виды расширений

| Тип                    | Пример    |
| ---------------------- | --------- |
| Подстановка пути       | `~`       |
| Подстановка переменных | `$USER`   |
| Подстановка команд     | `$(date)` |
| Шаблоны файлов         | `*.txt`   |
| Диапазоны              | `[a-z]`   |
| Фигурные скобки        | `{1..10}` |

# Задание 1. Подстановка домашнего каталога

## Требуется

Проверить работу символа:

```bash
~
```

---

# Решение

```bash
echo ~
```

---

# Проверка

Сравнить результат:

```bash
echo ~

echo $HOME
```

---

# Объяснение

Символ:

```bash
~
```

автоматически заменяется домашним каталогом пользователя.

---

# DevOps-ситуация

В скриптах практически никогда не пишут:

```bash
/home/user
```

Используют:

```bash
~
```

или

```bash
$HOME
```

чтобы скрипт работал у любого пользователя.

# Задание 3. Подстановка команд

## Требуется

Выполнить:

```bash
echo $(date)
```

---

# Решение

```bash
echo $(date)
```

---

# Затем

```bash
mkdir backup-$(date +%Y%m%d)
```

---

# Проверка

```bash
ls
```

---

# Объяснение

Сначала выполняется команда:

```bash
date +%Y%m%d
```

а её результат подставляется в команду `mkdir`.

---

# DevOps-ситуация

Так создаются резервные копии:

```bash
tar -czf backup-$(date +%F).tar.gz /etc
```

# Задание 5. Шаблон ?

Создать файлы:

```bash

touch file2.txt

```

---

# Выполнить

```bash

```

---

# Решение

```bash
ls file?.txt
```

---

# Объяснение

Символ:

```bash
?
```

означает:

```text
ровно один символ
```

# Основные конструкции урока

| Конструкция  | Назначение                     |
| ------------ | ------------------------------ |
| `~`          | Домашний каталог               |
| `$USER`      | Пользователь                   |
| `$HOME`      | Домашний каталог из переменной |
| `$(команда)` | Подстановка команды            |
| `*`          | Любое количество символов      |
| `?`          | Один символ                    |
| `[abc]`      | Один символ из набора          |
| `[0-9]`      | Диапазон символов              |
| `{a,b,c}`    | Список значений                |
| `{1..10}`    | Последовательность             |
