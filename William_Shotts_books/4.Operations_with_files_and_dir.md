# Теоретическая часть

## 1. Файловая система Linux

В Linux всё представлено в виде файлов:

- обычные файлы;
- каталоги;
- устройства;
- сокеты;
- ссылки.

Главная директория системы:

```bash
/
```

Она называется **корневой каталог**.

---

## 2. Основные каталоги Linux

| Каталог | Назначение                   |
| ------- | ---------------------------- |
| `/home` | Домашние папки пользователей |
| `/etc`  | Конфигурация системы         |
| `/var`  | Логи и изменяемые данные     |
| `/tmp`  | Временные файлы              |
| `/usr`  | Программы и библиотеки       |
| `/bin`  | Базовые команды Linux        |

---

# Абсолютные и относительные пути

## Абсолютный путь

Полный путь от корня `/`

Пример:

```bash
/home/student/docs/file.txt
```

---

## Относительный путь

Путь относительно текущей директории.

Пример:

```bash
docs/file.txt
```

# Основные команды

---

# Команда `mkdir`

## Назначение

Создание каталогов.

---

## Синтаксис

```bash
mkdir [опции] имя_каталога
```

---

## Пример

```bash
mkdir project
```

Создаст каталог:

```text
project/
```

---

## Создание вложенных каталогов

```bash
mkdir -p project/backend/api
```

Опция:

```bash
-p
```

создаёт всю структуру сразу.

---

# Команда `touch`

## Назначение

Создание пустых файлов.

---

## Пример

```bash
touch notes.txt
```

---

# Команда `cp`

## Назначение

Копирование файлов и каталогов.

---

## Синтаксис

```bash
cp источник назначение
```

---

## Пример копирования файла

```bash
cp file1.txt backup/
```

---

## Копирование каталогов

Для каталогов используется ключ:

```bash
-r
```

---

## Пример

```bash
cp -r project backup/
```

---

# Команда `mv`

## Назначение

- перемещение файлов;
- переименование файлов.

---

## Пример перемещения

```bash
mv file.txt docs/
```

---

## Пример переименования

```bash
mv old.txt new.txt
```

---

# Команда `rm`

## Назначение

Удаление файлов и каталогов.

---

## Удаление файла

```bash
rm file.txt
```

---

## Удаление каталога

```bash
rm -r project/
```

---

## Опасная команда

```bash
rm -rf
```

Ключи:

- `-r` — удалять рекурсивно;
- `-f` — без подтверждения.

---

# Команда `ln`

## Назначение

Создание ссылок.

---

# Символические ссылки (Soft Link)

Создаются так:

```bash
ln -s original.txt link.txt
```

---

## Особенности

- работают как ярлыки;
- могут ссылаться на каталог;
- ломаются при удалении оригинала.

---

# Жёсткие ссылки (Hard Link)

Создаются так:

```bash
ln original.txt hardlink.txt
```

---

## Особенности

- это второе имя файла;
- inode одинаковый;
- файл существует, пока есть хотя бы одна ссылка.

# Контрольные вопросы

## 1. Чем отличается `cp` от `mv`?

- `cp` копирует;
- `mv` перемещает или переименовывает.

---

## 2. Для чего используется `-r`?

Рекурсивная работа с каталогами.

---

## 3. Чем hard link отличается от soft link?

| Hard Link                         | Soft Link    |
| --------------------------------- | ------------ |
| Это ещё одно имя файла            | Это ярлык    |
| inode одинаковый                  | inode другой |
| Работает после удаления оригинала | Ломается     |

---

## 4. Что делает команда:

```bash
rm -rf folder
```

Полностью удаляет каталог без подтверждения.
