## Глава 8. Домашнее задание

Выполняй в терминале, фиксируй, какими комбинациями/подстановками ты решил каждый пункт. Задания идут от простого к сложному — не перенабирай команды руками там, где можно достать их из истории.

1. Выполни `pwd`, затем `ls -la`. Не нажимая стрелку «вверх», вернись к команде `pwd` и выполни её повторно через одну из комбинаций навигации по истории.

2. Начни печатать `cat /etc/host` и с помощью `Tab` посмотри, какие варианты дополнения предлагает оболочка. Допиши до `/etc/hostname` минимумом нажатий.

3. Набери длинную (но не выполняй её) строку `docker compose -f /opt/stack/docker-compose.prod.yml up -d` и одной комбинацией перенеси курсор в начало строки, чтобы дописать перед ней `sudo`, затем одной комбинацией снова прыгни в конец строки.

4. Выполни `echo "deploy version 1.4.2"`. Теперь повтори предыдущую команду целиком, но с `sudo` впереди, используя подстановку истории.

5. Создай файл `touch /tmp/report-$(date +%F).txt`, а затем выведи его содержимое (`cat`), переиспользовав последний аргумент предыдущей команды (не копируя путь).

6. Час назад (представим) ты запускал `journalctl -u nginx --since "10:00" --no-pager`. Найди эту команду через инкрементальный поиск, вытащи её в строку для редактирования (не выполняя сразу), поменяй `nginx` на `docker` и выполни.

7. Ты выполнил `kubectl scale deployment api --replicas=3 -n dev`. Краткой подстановкой повтори её, заменив `dev` на `prod`. Затем подумай и проверь: что произойдёт, если в команде слово `dev` встречается дважды?

8. Открой `history`, найди номер какой-нибудь своей команды из начала сессии и вызови её по номеру через `!номер`. Затем вызови последнюю по счёту команду, начинавшуюся со слова `cat`, не указывая её номер.

9. Набери ошибочную команду `systemctt restart nginx` (с опечаткой), но **не выполняй**. С помощью комбинаций редактирования (`Ctrl+W`, `Alt+B`, `Ctrl+T` и т. п.) исправь `systemctt` на `systemctl` без полного стирания строки. Зафиксируй, какие комбинации использовал.

10. Запиши сеанс терминала командой `script` в файл `/tmp/hw8-session.log`, выполни внутри него 3–4 любые команды из заданий выше, корректно заверши запись и убедись (`cat` или `less`), что и ввод, и вывод сохранились в файл.

11. (со звёздочкой) По умолчанию bash хранит 500 команд истории (см. главу 8). Разберись самостоятельно, какие переменные окружения управляют размером истории, форматом времени и дедупликацией команд, и предложи, какие строки ты бы добавил в `~/.bashrc`, чтобы: хранить 10000 команд, показывать дату/время рядом с каждой записью в `history` и не засорять историю дубликатами подряд идущих команд.
